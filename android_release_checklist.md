# Android Release 檢查清單（Flutter App）

本文件用於每次 Android 版本釋出前的標準驗收，目標是確保：

1. 功能可用（登入、掃描、上傳、通知、狀態收斂）
2. 可靠性可接受（網路波動、重試、斷點續傳）
3. 風險可控（權限、錯誤可觀測、無阻塞型問題）

---

## 0. 使用方式（每次發版都做）

- 建議在 **至少 2 台 Android 裝置**（不同品牌/版本）執行。
- 每個版本至少跑 1 次完整流程（含 50+ 張照片與至少 1 個大檔案）。
- 每次驗收都保留一份測試紀錄（日期、機型、版本、結果、阻塞問題）。

---

## 1. 環境與建置檢查（Preflight）

- [ ] `flutter pub get` 成功
- [ ] `flutter analyze` 無 error（warning 可追蹤）
- [ ] `flutter test` 既有測試全過
- [ ] Android Debug build 可安裝、可啟動
- [ ] Android Release build 可產出（APK/AAB）
- [ ] App 啟動後無立即崩潰

---

## 2. 帳號與權限檢查

- [ ] Google 登入成功（首次）
- [ ] 重開 App 後可靜默登入（若未過期）
- [ ] 相簿讀取權限流程正常（允許/拒絕皆有正確提示）
- [ ] Android 通知權限流程正常（允許/拒絕皆有可理解行為）
- [ ] 拒絕通知權限時，上傳流程仍可執行（僅少通知）

---

## 3. 核心功能流程驗收（E2E）

### 3.1 掃描與資料建立
- [ ] 可讀取本地相簿並選取
- [ ] 掃描完成後 Dashboard 顯示相簿統計
- [ ] 重複掃描同一批資料不會爆量重複（防重複策略有效）

### 3.2 上傳與相簿建立
- [ ] 點擊「開始備份」後可進入處理流程
- [ ] 小量資料（10 張）可全部完成
- [ ] Google Photos 端可看到相簿與媒體
- [ ] DB 狀態有收斂（`uploading` 不會永久卡住）

### 3.3 進度與完成通知
- [ ] 上傳中有進度通知（已處理/總數）
- [ ] 完成後有摘要通知（成功/失敗/待重試）
- [ ] 無通知權限時不崩潰，且流程不被阻塞

---

## 4. 可靠性與異常情境

### 4.1 網路波動
- [ ] 上傳中切換 Wi-Fi/行動網路，流程可恢復
- [ ] 短暫斷網後可將可重試項目回到 `pending`
- [ ] 不會因單一錯誤導致整批中止

### 4.2 大檔案與續傳
- [ ] 至少 1 個大檔案（建議 200MB+）可上傳成功
- [ ] 大檔案期間記憶體不異常飆升（無明顯 OOM 徵兆）

### 4.3 限流與重試
- [ ] 遇到 429 時不會整批失敗
- [ ] 有退避/重試行為，最終狀態合理（done/failed/pending）

---

## 5. UI/UX 與可操作性檢查

- [ ] 文案與實際行為一致（不誤導背景能力）
- [ ] 錯誤訊息可理解（非純技術堆疊）
- [ ] 設定中的危險操作（清除/重置）可確認且可回復
- [ ] Dashboard 進度、失敗數、完成數可讀性良好

---

## 6. 可觀測性與除錯資料

- [ ] 錯誤日誌可查看最近紀錄
- [ ] 關鍵節點有 log（token、sessionUrl、upload、batchCreate、狀態轉移）
- [ ] 發生問題時可從 log 還原流程中斷位置

建議每次驗收保存：

- App 版本號
- 測試機型與 Android 版本
- 問題發生時間點與錯誤日誌片段

---

## 7. 發版決策（Go / No-Go）

### Go 條件（全部成立）
- [ ] 核心 E2E 流程可完成
- [ ] 大檔案測試通過
- [ ] 通知與狀態收斂正常
- [ ] 無 P0 / P1 阻塞缺陷

### No-Go 條件（任一成立即暫停發版）
- [ ] 點擊上傳後卡住無進度且無法恢復
- [ ] `uploading` 狀態長時間不收斂
- [ ] Google 端未建立相簿/未入檔但本地顯示完成
- [ ] 大量失敗且無重試或回復策略
