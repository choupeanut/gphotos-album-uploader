# 關於 Google Cloud Console 上的「測試使用者」

如果您在設定 **OAuth 同意畫面 (OAuth consent screen)** 時沒有看到「測試使用者 (Test users)」的設定區塊，通常是因為以下兩種情況：

### 情況一：您的專案狀態被設為了「發布 (In production)」
如果您在設定 OAuth 同意畫面時，不小心將「發布狀態 (Publishing status)」按到了 **發布 (Publish app)**，那麼它就不會再顯示測試使用者。
*   **影響**：因為 Google Photos 的 API 屬於敏感範圍 (Sensitive scope)，如果您的 App 是「In production」狀態，登入時會跳出警告畫面說「此應用程式尚未通過 Google 驗證」，使用者需要點擊「進階 > 繼續前往」才能登入。
*   **解決方式**：這不會影響您自己的測試。您可以直接用手機登入，遇到警告時點擊「繼續」即可。或者，您可以在 OAuth 同意畫面的首頁，點擊 **「返回測試階段 (Back to testing)」**。切回測試階段後，「測試使用者」區塊就會重新出現。

### 情況二：您的專案是綁定在 Google Workspace (G Suite) 內部
如果您是用公司或學校的 Google 帳號建立這個專案，並且在建立 OAuth 同意畫面時選擇了 **「內部 (Internal)」**。
*   **影響**：內部專案**不需要**設定測試使用者。任何屬於該公司/學校網域的 Google 帳號都可以直接登入並測試。
*   **解決方式**：如果是這種情況，您不需要做任何設定，直接拿您的公司/學校帳號在手機上登入即可。

---

### 下一步該怎麼做？
既然您已經設定好了，請直接在您的手機上安裝 APK 並嘗試登入：
1. 點擊「使用 Google 帳號登入」。
2. 如果順利跳出選擇帳號的畫面，請選擇您的帳號。
3. 如果出現「尚未驗證」的警告，點「進階」然後「繼續」。
4. 授權頁面會詢問是否允許存取 Google 相片，點擊允許。

如果登入成功且進入到 Dashboard，就代表整個 API 串接已經大功告成！您可以嘗試按下「步驟 1：掃描本地相簿」來看看它是不是能順利找出您的照片！
